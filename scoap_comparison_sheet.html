SCOAP vs the Min-Max Parallel-Time Heuristic

Reference sheet for comparing the original sequential SCOAP rules with the classroom rule: "use max when all required inputs must be ready, use min when any one input is enough."

What each metric is measuring

Metric What it measures Core evaluation rule
Combinational SCOAP
CC0, CC1		 Estimated control effort through a combinational network. Evaluate a justifying input assignment by summing the needed input costs, then add one gate level.
Sequential SCOAP
SC0, SC1		 Estimated control effort in a sequential circuit. Same justification logic as SCOAP, but combinational gates add no extra +1. The extra +1 appears when a clocked storage step is needed.
Min-Max heuristic
T0, T1		 Earliest time frame if independent input conditions can be prepared in parallel. For one justification, the finishing time is controlled by the slowest required input, so use max. Then take min across alternative justifications.

Why the rules split

SCOAP is a cost metric, so when a gate requires multiple input conditions together, the original rule adds their costs.

Min-Max is a parallel scheduling metric, so when multiple input conditions can be prepared at the same time, the delay is the slowest one, not the sum.

For one output value v and a justifying assignment j:
SCOAP: cost(j) = sum of required input controllabilities
Min-Max: time(j) = max of required input times

Then both methods choose the best justification by taking min across the possible justifications.

How to read the AND / OR rules

All inputs required for the target output:

SCOAP uses sum

Min-Max uses max

Any one input enough for the target output:

Both use min

Storage element note

In sequential SCOAP, the extra +1 does not come from a combinational gate. It comes from a required clocked state transition through storage.

That is why sequential formulas look like combinational formulas with the gate-level +1 removed.

Z example from your figure

Gate Z is an AND with inputs [0,1] and [1,2].

Sequential SCOAP: SC0(Z) = min(0,1) = 0

Sequential SCOAP: SC1(Z) = 1 + 2 = 3

Min-Max heuristic: T1(Z) = max(1,2) = 2

So 3 and 2 are both valid, but they belong to different metrics.

Where the two rule systems come from

Rule family Origin and interpretation
Original SCOAP rules Goldstein defines controllability by evaluating justifying input assignments using the sum of the required input controllabilities, then taking the best one. For sequential SCOAP, combinational cells have sequential depth 0, so there is no per-gate +1 there.
Min-Max method This comes from a parallel arrival-time or time-frame interpretation. If several input conditions are all needed and can be prepared simultaneously, the total time is governed by the slowest required input, hence max. If only one input condition is enough, use min.
